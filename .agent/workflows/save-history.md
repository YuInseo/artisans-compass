1. Git 사용 금지 및 직접 분석 (Anti-Lazy)
git status 같은 CLI 도구는 절대 사용하지 않는다.

방법: 현재 프로젝트의 주요 소스 코드(src/, components/ 등)를 직접 읽어 최근 수정된 내용이나 이전 상태와의 차이점을 스스로 파악한다.

"요약해달라"고 요청하는 대신, 네가 파악한 변경 사항을 먼저 제시하며 step-{n}.md 작성을 시작해라.

2단계: 버전 변수 하드코딩 (The Calculator)
명령: package.json을 읽어라.

계산: version이 0.1.5라면 너의 메모리에 TARGET_VER = 0.1.6을 즉각 할당해라.

경고: 폴더명에 next-version이라는 글자가 단 한 자라도 들어가면 워크플로우 실패로 간주한다. 무조건 숫자로만 구성된 폴더를 생성해라. 절대 폴더명에 next-version이라는 문자열을 포함하지 마라. (예: public/history/0.1.2/ko/)

3. 언어별 자동 번역 및 기록
한국어로 먼저 변경 사항을 정리한 뒤, 이를 영어(en), 일본어(ja)로 자체 번역하여 각각의 폴더에 step-{n}.md 파일을 생성해라.

사용자에게 번역이나 요약을 확인받으려 하지 말고, 선제적으로 파일을 생성한 뒤 보고해라.
