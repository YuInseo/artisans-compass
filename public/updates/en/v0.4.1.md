# Update Notice (v0.4.1)
- **Date**: 2026-02-26
- **Description**:

## 0.4.1 (Quotes & Reminders UI Update)
- Upgraded the UI for the Custom Quotes (`QuoteManager`) and Reminders (`ReminderManager`) to a more premium design (glassmorphism effect, optimized spacing, better element placement).


## 0.4.1 (Quotes & Reminders Edit)
- Added an edit feature to Custom Quotes (`QuoteManager`) and Reminders (`ReminderManager`). You can now conveniently edit the text of existing items directly inline and save it.
- Further increased the padding in the input fields and list items to completely fixed the issue where the text was cut off on the left side of the modal.
- Fixed a layout issue where the entire `Settings Modal` content was slightly clipped on the left by adjusting the padding of the scrolling area container.
- Removed the inline editing functionality for quotes directly within the widget and dashboard view.
- Improved the logic so that your "Custom Quotes" will now appear prioritized over the default quotes in both the widget and the morning Inspiration Modal.


## 0.4.1 (Quotes & Reminders Edit)
- Added a `Show Routines` ON/OFF toggle in the Timetable settings popover located in the Widget and Dashboard view, allowing users to show or hide their repetitive routines.
- This setting is linked to the global AppSettings and will instantly reflect in the rendered daily graph (`TimeTableGraph`).


## 0.4.1 (Quotes & Reminders Edit)
- Significantly reduced the heavy rendering lag (stuttering) that occurred when transitioning from Widget Mode back to General Mode.
- Removed a rendering bottleneck by changing the dynamic async theme imports to static synchronous imports.
- Reordered the IPC communication so that the OS window expands first, followed by a 50ms delay before rendering the heavy main UI, preventing window-resize and React DOM updates from clashing.


## 0.4.1 (Quotes & Reminders Edit)
- Polished the spacing between the 'Daily Quest' and 'Goal Setting' buttons at the bottom of the sidebar.
- Grouped both buttons into a single flex container with a consistent gap (`gap-2`), giving the UI a cleaner and more cohesive look.
